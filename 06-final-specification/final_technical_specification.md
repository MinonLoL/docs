О документе: итоговая техническая спецификация MVP на основе реализованного кода и документации.

---

## Область решения

MVP включает Flask-приложение с Jinja2-шаблонами, SQLite-БД и набором REST API эндпоинтов. Развёртывание — локальное или через nginx + pm2 на VPS.

## Функциональные модули

- Аутентификация и разграничение доступа по ролям (operator / supervisor / analyst)
- Справочник ПВЗ и расписаний
- Регистрация операций с 5+ валидациями
- Отчёт загрузки с фильтрами (период / ПВЗ / тип)
- KPI-панель (средняя загрузка, доля перегрузок, критические перегрузки)
- Тепловая карта (день × час, цветовая шкала по порогам нагрузки)
- Лог ошибок валидации
- Экспорт CSV (роль analyst)

## Нефункциональные требования

- Role-based access на уровне backend (декоратор `@role_required`).
- Все запросы возвращают JSON с кодами 200 / 201 / 400 / 401 / 403 / 404.
- Кодировка всех файлов и CSV — UTF-8.
- Построение отчёта по ≥200 операциям — интерактивно (без пагинации).
- Корректная обработка «пустых часов» (0 операций в час → load = 0).
