# DOC-REQ-004 — MoSCoW-матрица требований

| Версия | Статус | Дата создания | Дата обновления |
|--------|--------|---------------|-----------------|
| v1.0   | Draft  | 2026-05-01    | 2026-05-19      |

О документе: приоритизация функциональных требований по методу MoSCoW.

Для кого: команда проекта, заказчик.

Основано на: [user stories](DOC-REQ-001-user-stories.md).

---

## Must Have — критически важно

| Требование                                              | Связанные US         |
|---------------------------------------------------------|----------------------|
| Авторизация и разграничение доступа по ролям            | US-01                |
| Просмотр информации о ПВЗ и расписании                  | US-02                |
| Регистрация операций `in` / `out` / `return`            | US-03                |
| Валидация операций (5+ правил) с логированием ошибок    | US-03                |
| Список последних операций                               | US-04                |
| Список ПВЗ региона для супервайзера                     | US-07                |
| Фильтрация операций по дате, ПВЗ, типу                  | US-08                |
| Отчёт загрузки: `ops`, `capacity`, `load %`, `overload` | US-09                |
| Доступ аналитика ко всем данным без ограничений         | US-12                |
| KPI-панель: средняя загрузка, доля перегрузок           | US-13                |
| Экспорт CSV: `pvz_id`, `date`, `hour`, `ops`, `capacity`, `load` | US-14    |

## Should Have — важно, но не блокирует

| Требование                                              | Связанные US         |
|---------------------------------------------------------|----------------------|
| Виджет текущей загрузки (load % текущего часа)          | US-05                |
| Просмотр лога ошибок валидации                          | US-11                |
| Тепловая карта для любого ПВЗ региона (супервайзер)     | US-10                |

## Could Have — желательно

| Требование                                              | Связанные US         |
|---------------------------------------------------------|----------------------|
| Тепловая карта для оператора (своя точка)               | US-06                |
| Ручное создание операций аналитиком (массовая загрузка) | US-15                |
| Дашборд с пороговыми индикаторами                       | —                    |

## Won't Have — вне scope

| Требование                                              | Обоснование                              |
|---------------------------------------------------------|------------------------------------------|
| Интеграция с реальными геосервисами                     | Учебный проект, нет реальных данных      |
| Подключение к терминалам/сканерам                       | Учебный проект                           |
| Push-уведомления о перегрузке в реальном времени        | Сложность реализации вне MVP             |
| Прогнозирование нагрузки (ML)                           | За рамками задания                       |
| Интеграция с BI-системами (Tableau, Power BI)           | За рамками задания                       |
