## Сценарий 1: Оператор регистрирует операцию

**Роль:** Оператор ПВЗ  
**Цель:** Зафиксировать факт приёма/выдачи/возврата посылки.

| Шаг | Действие пользователя           | Реакция системы                                                      |
|-----|---------------------------------|----------------------------------------------------------------------|
| 1   | Открывает раздел «Операции»     | Отображается форма с полями: тип, дата/время (автозаполнение — сейчас)|
| 2   | Выбирает тип операции           | Подсвечивается выбранный тип (`in` / `out` / `return`)               |
| 3   | Нажимает «Зарегистрировать»     | Система валидирует: ПВЗ открыт, тип корректен, время в расписании    |
| 4а  | (успех)                         | Операция сохранена, появляется уведомление, счётчик часа обновляется |
| 4б  | (ошибка)                        | Сообщение с причиной отказа (напр. «ПВЗ закрыт в это время»)        |

**Альтернативный поток:** Если тип не выбран — кнопка «Зарегистрировать» недоступна.

## Сценарий 2: Оператор смотрит загрузку своей точки

**Роль:** Оператор ПВЗ  
**Цель:** Понять, насколько близок предел мощности в текущем часе.

| Шаг | Действие пользователя              | Реакция системы                                                       |
|-----|------------------------------------|-----------------------------------------------------------------------|
| 1   | Открывает «Мою загрузку»           | Отображается карточка: текущий час, `ops`, `capacity`, `load %`      |
| 2   | Прокручивает вниз                  | Таблица загрузки по часам за сегодня                                  |
| 3   | Нажимает на конкретный час         | Список операций за этот час с типами                                  |

## Сценарий 3: Супервайзер просматривает сводный отчёт региона

**Роль:** Супервайзер региона  
**Цель:** Выявить перегруженные точки и принять управленческое решение.

| Шаг | Действие пользователя              | Реакция системы                                                               |
|-----|------------------------------------|-------------------------------------------------------------------------------|
| 1   | Открывает «Отчёт загрузки»         | Сводная таблица по всем ПВЗ региона: `pvz_id`, `ops`, `capacity`, `load %`   |
| 2   | Устанавливает фильтр по дате       | Таблица пересчитывается для выбранного периода                                |
| 3   | Видит строки с красной подсветкой  | Это точки с `load > 100%` (флаг `overload = true`)                           |
| 4   | Кликает по ПВЗ в таблице           | Раскрывается почасовой отчёт по этому ПВЗ                                    |

## Сценарий 4: Супервайзер просматривает лог ошибок

**Роль:** Супервайзер региона  
**Цель:** Выявить нарушения — операции, зафиксированные вне рабочих часов.

| Шаг | Действие пользователя            | Реакция системы                                                     |
|-----|----------------------------------|---------------------------------------------------------------------|
| 1   | Открывает «Лог ошибок»           | Таблица: `pvz_id`, `ts`, `type`, `reason`, `logged_at`             |
| 2   | Фильтрует по ПВЗ и дате          | Список сужается до выбранных параметров                             |
| 3   | Нажимает на строку               | Детали инцидента: точный момент, тип операции, причина отклонения   |

## Сценарий 5: Аналитик экспортирует CSV

**Роль:** Операционный аналитик  
**Цель:** Выгрузить данные для анализа в Excel / Python.

| Шаг | Действие пользователя                   | Реакция системы                                                        |
|-----|-----------------------------------------|------------------------------------------------------------------------|
| 1   | Открывает «Экспорт»                     | Форма с фильтрами: период, ПВЗ, тип операции                          |
| 2   | Устанавливает период и нажимает «Экспорт»| Генерируется CSV файл, начинается загрузка                            |
| 3   | Открывает файл                          | Столбцы: `pvz_id`, `date`, `hour`, `ops`, `capacity`, `load`; UTF-8   |

**Альтернативный поток:** Если за выбранный период данных нет — сообщение «Нет данных для экспорта».
