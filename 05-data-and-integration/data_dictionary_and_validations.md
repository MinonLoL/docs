## Словарь данных

### Таблица `pvz`

| Поле               | Тип     | Nullable | Уникальность | Допустимые значения    | Пример                      |
|--------------------|---------|----------|--------------|------------------------|-----------------------------|
| `pvz_id`           | INTEGER | No       | Yes (PK)     | > 0                    | 1                           |
| `address`          | TEXT    | No       | No           | Непустая строка        | «ул. Ленина, 10, Москва»   |
| `capacity_per_hour`| INTEGER | No       | No           | > 0                    | 50                          |
| `region`           | TEXT    | No       | No           | Непустая строка        | «Центральный»               |

### Таблица `schedule`

| Поле        | Тип     | Nullable | Уникальность           | Допустимые значения       | Пример  |
|-------------|---------|----------|------------------------|---------------------------|---------|
| `id`        | INTEGER | No       | Yes (PK)               | > 0                       | 1       |
| `pvz_id`    | INTEGER | No       | No                     | ∈ pvz.pvz_id              | 1       |
| `weekday`   | INTEGER | No       | Unique (pvz_id+weekday)| 0 (пн) … 6 (вс)          | 0       |
| `open_time` | TEXT    | No       | No                     | HH:MM, 00:00 – 23:59      | «09:00» |
| `close_time`| TEXT    | No       | No                     | HH:MM; > open_time        | «21:00» |

### Таблица `operations`

| Поле     | Тип     | Nullable | Уникальность | Допустимые значения         | Пример              |
|----------|---------|----------|--------------|-----------------------------|---------------------|
| `op_id`  | INTEGER | No       | Yes (PK)     | > 0                         | 1001                |
| `pvz_id` | INTEGER | No       | No           | ∈ pvz.pvz_id                | 2                   |
| `ts`     | TEXT    | No       | No           | ISO 8601, datetime          | «2026-05-12T10:23» |
| `type`   | TEXT    | No       | No           | `in` \| `out` \| `return`  | «out»               |

### Таблица `error_log`

| Поле       | Тип     | Nullable | Допустимые значения           | Пример                          |
|------------|---------|----------|-------------------------------|---------------------------------|
| `id`       | INTEGER | No       | > 0                           | 55                              |
| `pvz_id`   | INTEGER | No       | любой (может быть невалидным) | 99                              |
| `ts`       | TEXT    | No       | ISO 8601                      | «2026-05-12T07:00»             |
| `op_type`  | TEXT    | No       | любое значение из запроса     | «in»                            |
| `reason`   | TEXT    | No       | строка с описанием            | «pvz_id не найден»              |
| `logged_at`| TEXT    | No       | ISO 8601, datetime            | «2026-05-12T10:30:00»          |

## Правила валидации API

### POST /api/operations — создание операции

| № | Поле       | Правило                                                                  | Ошибка при нарушении               |
|---|------------|--------------------------------------------------------------------------|------------------------------------|
| V1| `pvz_id`   | Должен существовать в таблице `pvz`                                      | 400: «pvz_id не найден»            |
| V2| `type`     | Должен быть одним из: `in`, `out`, `return`                              | 400: «Недопустимый тип операции»   |
| V3| `ts`       | День недели из `ts` должен иметь запись в `schedule` для данного `pvz_id`| 400: «ПВЗ не работает в этот день» |
| V4| `ts`       | Время из `ts` должно быть в диапазоне [open_time, close_time]            | 400: «Операция вне рабочих часов»  |
| V5| Общее      | Запрос должен содержать все три поля: `pvz_id`, `ts`, `type`             | 400: «Отсутствует обязательное поле»|

### Валидации данных расписания (при инициализации)

| № | Правило                                          | Описание                              |
|---|--------------------------------------------------|---------------------------------------|
| V6| `weekday ∈ {0, 1, 2, 3, 4, 5, 6}`               | Недопустимые дни недели отвергаются   |
| V7| `open_time < close_time`                         | Инвертированное расписание недопустимо|
| V8| `capacity_per_hour > 0`                          | Нулевая или отрицательная мощность — ошибка|

## Расчёт нагрузки (агрегации)

| Метрика          | Формула                                               | Примечание                               |
|------------------|-------------------------------------------------------|------------------------------------------|
| `ops_per_hour`   | COUNT(op_id) GROUP BY pvz_id, DATE(ts), strftime('%H', ts) | Все типы операций суммируются         |
| `load`           | ops_per_hour / capacity_per_hour                      | Значение от 0.0 до ~2.0 и выше           |
| `load_pct`       | load × 100                                            | В процентах                              |
| `overload`       | load > 1.0                                            | Булев флаг для подсветки в отчёте       |
| `peak_avg_load`  | AVG(load) WHERE hour ∈ «часы пик»                     | Определяется настройками бизнеса (9–21) |
| `overload_share` | COUNT(overload=True) / COUNT(*) × 100                 | Доля перегруженных интервалов, %        |
