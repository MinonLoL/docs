## Подготовка к демонстрации

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Сгенерировать синтетические данные (≥200 операций)
python generate_realistic_data.py

# 3. Запустить приложение
python app.py

# 4. Открыть браузер
# http://127.0.0.1:5000
```

**Тестовые учётные записи:**

| Роль      | Логин       | Пароль   | pvz_id |
|-----------|-------------|----------|--------|
| Оператор  | operator1   | pass123  | 1      |
| Супервайзер | supervisor | pass123  | —      |
| Аналитик  | analyst     | pass123  | —      |

## TC-001: Авторизация и разграничение доступа

- **Предусловие:** приложение запущено.
- **Шаги:** войти как `operator1` → открыть список ПВЗ.
- **Ожидаемо:** видны данные только ПВЗ с `pvz_id=1`.
- **Проверка ролей:** войти как `supervisor` → видны все ПВЗ региона; войти как `analyst` → видны все ПВЗ.

---

## TC-002: Регистрация корректной операции

- **Предусловие:** вход как `operator1`; текущее время в рабочих часах ПВЗ.
- **Шаги:** перейти в «Операции» → выбрать тип `out` → нажать «Зарегистрировать».
- **Ожидаемо:** операция сохранена, появляется в списке; счётчик часа обновляется.

---

## TC-003: Отклонение операции вне рабочих часов

- **Предусловие:** попытка создать операцию с `ts` в нерабочее время.
- **Шаги:** POST `/api/operations` с `ts="2026-05-12T03:00:00"` для ПВЗ, работающего с 09:00.
- **Ожидаемо:** ответ 400 Bad Request; запись появляется в `error_log` с reason `«Операция вне рабочих часов»`.

---

## TC-004: Отклонение операции с неверным типом

- **Шаги:** POST `/api/operations` с `type="delivery"`.
- **Ожидаемо:** ответ 400; `error_log` содержит `reason="Недопустимый тип операции"`.

---

## TC-005: Отклонение операции для несуществующего ПВЗ

- **Шаги:** POST `/api/operations` с `pvz_id=9999`.
- **Ожидаемо:** ответ 400; `error_log` содержит `reason="pvz_id не найден"`.

---

## TC-006: Отчёт загрузки ПВЗ

- **Предусловие:** в БД есть операции за последние 7 дней.
- **Шаги:** открыть «Отчёт загрузки» → установить период → нажать «Применить».
- **Ожидаемо:** таблица с колонками `pvz_id`, `date`, `hour`, `ops`, `capacity`, `load %`; строки с `load > 100%` подсвечены красным.

---

## TC-007: Фильтрация отчёта

- **Шаги:** в отчёте загрузки установить фильтр по `pvz_id=1` и типу `out` → применить.
- **Ожидаемо:** в таблице только строки для ПВЗ №1, учитываются только операции типа `out`.

---

## TC-008: Тепловая карта

- **Шаги:** открыть «Тепловая карта» → выбрать ПВЗ → выбрать период.
- **Ожидаемо:** матрица «день × час» с цветовой шкалой; зелёный (<85%), жёлтый (85–100%), красный (>100%).

---

## TC-009: KPI-панель

- **Шаги:** открыть «Аналитика» → выбрать период.
- **Ожидаемо:** виджеты: средняя загрузка в пик, доля перегруженных интервалов (%), число критических перегрузок.

---

## TC-010: Экспорт CSV

- **Предусловие:** вход как `analyst`.
- **Шаги:** открыть «Экспорт» → выбрать период → нажать «Скачать CSV».
- **Ожидаемо:** загружается файл `report.csv`; содержит строки `pvz_id,date,hour,ops,capacity,load`; кодировка UTF-8.

---

## TC-011: Лог ошибок

- **Предусловие:** в системе есть записи ошибок валидации.
- **Шаги:** открыть «Лог ошибок» → применить фильтр по ПВЗ.
- **Ожидаемо:** таблица с полями `pvz_id`, `ts`, `type`, `reason`, `logged_at`.
