## Сценарий 1: Авторизация оператора и ограничение данных

- **Предусловие:** пользователь `operator1` с ролью `operator` и `pvz_id=1`.
- **Шаги:** войти → открыть GET /api/pvz.
- **Ожидаемо:** возвращается только ПВЗ с `pvz_id=1`; другие точки недоступны.

## Сценарий 2: Создание корректной операции

- **Предусловие:** ПВЗ №1 работает по понедельникам с 09:00 до 21:00.
- **Шаги:** POST /api/operations `{"pvz_id":1,"ts":"2026-05-18T10:00:00","type":"out"}`.
- **Ожидаемо:** ответ 201; операция появляется в GET /api/operations.

## Сценарий 3: Валидация — нерабочее время

- **Шаги:** POST /api/operations `{"pvz_id":1,"ts":"2026-05-18T02:00:00","type":"in"}`.
- **Ожидаемо:** ответ 400; в error_log добавлена запись с reason `«Операция вне рабочих часов»`.

## Сценарий 4: Валидация — недопустимый тип

- **Шаги:** POST /api/operations `{"pvz_id":1,"ts":"2026-05-18T10:00:00","type":"unknown"}`.
- **Ожидаемо:** ответ 400; reason `«Недопустимый тип операции»`.

## Сценарий 5: Валидация — несуществующий ПВЗ

- **Шаги:** POST /api/operations `{"pvz_id":9999,"ts":"2026-05-18T10:00:00","type":"in"}`.
- **Ожидаемо:** ответ 400; reason `«pvz_id не найден»`.

## Сценарий 6: Валидация — нет расписания на день

- **Предусловие:** ПВЗ №2 не работает в воскресенье.
- **Шаги:** POST /api/operations `{"pvz_id":2,"ts":"2026-05-17T10:00:00","type":"return"}` (воскресенье).
- **Ожидаемо:** ответ 400; reason `«ПВЗ не работает в этот день»`.

## Сценарий 7: Отчёт загрузки с расчётом load

- **Предусловие:** в БД есть операции за 2026-05-12.
- **Шаги:** GET /api/report/load?date_from=2026-05-12&date_to=2026-05-12.
- **Ожидаемо:** ответ 200; каждая строка содержит `ops_per_hour`, `capacity_per_hour`, `load_pct`; `load_pct = ops_per_hour / capacity_per_hour * 100`.

## Сценарий 8: Флаг перегрузки в отчёте

- **Предусловие:** в БД есть час, когда `ops > capacity`.
- **Ожидаемо:** в строке отчёта `overload = true`; в UI строка выделена красным.

## Сценарий 9: Фильтрация отчёта

- **Шаги:** GET /api/report/load?pvz_id=1&type=out.
- **Ожидаемо:** отчёт содержит только строки для ПВЗ №1, учитываются только операции `out`.

## Сценарий 10: Экспорт CSV

- **Предусловие:** роль `analyst`.
- **Шаги:** GET /api/report/export/csv?date_from=2026-05-01&date_to=2026-05-19.
- **Ожидаемо:** ответ 200 с `Content-Disposition: attachment`; файл содержит заголовок `pvz_id,date,hour,ops,capacity,load`; кодировка UTF-8.

## Сценарий 11: Запрет экспорта CSV для оператора

- **Предусловие:** роль `operator`.
- **Шаги:** GET /api/report/export/csv.
- **Ожидаемо:** ответ 403 Forbidden.

## Сценарий 12: KPI-панель

- **Шаги:** GET /api/report/kpi?date_from=2026-05-01&date_to=2026-05-19.
- **Ожидаемо:** ответ 200 с полями `avg_peak_load_pct`, `overload_share_pct`, `critical_overload_count`, `total_operations`.
