## Персона 1: Оператор ПВЗ

| Поле       | Значение                                                                                                                                   |
|------------|--------------------------------------------------------------------------------------------------------------------------------------------|
| Имя        | Иванова Марина Сергеевна                                                                                                                   |
| Роль       | Оператор ПВЗ                                                                                                                               |
| Описание   | Сотрудник конкретного пункта выдачи. Работает в смену, обслуживает клиентов и фиксирует операции приёма/выдачи/возврата.                   |
| Цели       | Быстро зарегистрировать операцию; видеть, сколько операций осталось до предела мощности; понимать, когда ожидается пик нагрузки.           |
| Боли       | Нет инструмента для оперативного контроля загрузки; очереди копятся незаметно; непонятно, когда взять перерыв, чтобы не создать затор.     |
| Ожидания   | Простой интерфейс для фиксации операций; визуальный индикатор текущей загрузки точки; просмотр графика за свою смену.                     |

## Персона 2: Супервайзер региона

| Поле       | Значение                                                                                                                                          |
|------------|---------------------------------------------------------------------------------------------------------------------------------------------------|
| Имя        | Петров Андрей Дмитриевич                                                                                                                          |
| Роль       | Супервайзер региона                                                                                                                               |
| Описание   | Менеджер, ответственный за группу ПВЗ в регионе. Контролирует показатели, распределяет ресурсы, реагирует на перегрузки.                         |
| Цели       | Видеть сводную загрузку всех ПВЗ региона; оперативно выявлять перегруженные точки; анализировать паттерны нагрузки по дням и часам.              |
| Боли       | Информация о загрузке разрозненна; реакция на перегрузку запаздывает на 1–2 часа; нет единой картины по региону в режиме реального времени.      |
| Ожидания   | Сводный отчёт с индикацией перегруженных точек; тепловая карта по часам/дням; лог нарушений (операции вне расписания).                          |

## Персона 3: Операционный аналитик

| Поле       | Значение                                                                                                                                      |
|------------|-----------------------------------------------------------------------------------------------------------------------------------------------|
| Имя        | Соколова Екатерина Игоревна                                                                                                                   |
| Роль       | Операционный аналитик                                                                                                                         |
| Описание   | Специалист по данным. Анализирует эффективность всей сети ПВЗ, строит отчёты, выгружает данные для внешнего анализа.                         |
| Цели       | Получить полный доступ к данным всех ПВЗ; рассчитать ключевые KPI; экспортировать данные в CSV для глубокого анализа в сторонних инструментах.|
| Боли       | Нет централизованного хранилища данных; выгрузка возможна только вручную; нет стандартизированного формата отчёта.                            |
| Ожидания   | Фильтрация по дате / ПВЗ / типу операции; экспорт CSV (pvz_id, date, hour, ops, capacity, load); KPI-панель с агрегатами по сети.            |
