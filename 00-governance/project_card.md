**PVZ Monitor** — система мониторинга загрузки пунктов выдачи заказов.

## 4. Цель проекта

Создать систему мониторинга загрузки ПВЗ: фиксация операций (приём/выдача/возврат), расчёт нагрузки относительно паспортной пропускной способности, формирование отчёта «Загрузка ПВЗ» с фильтрами и экспортом в CSV.

## 5. Проблема

Отсутствует централизованная система мониторинга нагрузки ПВЗ. Менеджеры не видят текущую загрузку в разрезе часов и дней, не могут оперативно реагировать на перегрузки (load > 100%) и эффективно планировать смены операторов.

## 6. Задачи

1. Смоделировать работу ПВЗ: расписание, пропускная способность, операции.
2. Реализовать 5+ бизнес-валидаций (ссылочная целостность, часы работы, тип операции и др.).
3. Построить отчёт загрузки: `ops_per_hour`, `capacity_per_hour`, `load %`, флаг перегрузки.
4. Реализовать фильтры по дате / ПВЗ / типу операции.
5. Обеспечить экспорт отчёта в CSV.
6. Разграничить доступ по ролям (оператор / супервайзер / аналитик).

## 7. KPI

| Метрика                              | Формула                                | Цель         |
|--------------------------------------|----------------------------------------|--------------|
| Загрузка в часы пик                  | ops_per_hour / capacity_per_hour × 100 | < 85%        |
| Доля интервалов с перегрузкой        | интервалы с load > 100% / все          | ≤ 5%         |
| Критическая перегрузка               | load > 120%                            | 0 интервалов |
| Объём данных                         | количество операций                    | ≥ 200        |
| Количество валидаций                 | —                                      | ≥ 5          |
