## Правила работы

1. **Ветвление**: каждая задача выполняется в отдельной ветке вида `feature/название-задачи`.
2. **Pull Request**: любой код мерджится только через PR с ревью минимум 1 участника.
3. **main**: ветка `main` используется только для финальных версий после контрольных точек.
4. **Коммиты**: осмысленные сообщения на русском или английском языке.
5. **Стендапы**: краткая синхронизация в Telegram-чате перед каждым спринтом.
6. **Документация**: обновляется параллельно с кодом; каждая КТ сопровождается актуализацией `docs/`.
7. **Дедлайны**: задачи фиксируются с датой завершения; просрочка обсуждается на ближайшем стендапе.

## Процесс работы с задачами

1. Задача создаётся в трекере (GitHub Issues / Kaiten).
2. Назначается ответственный участник.
3. Ответственный создаёт ветку и работает над задачей.
4. По готовности — PR с описанием изменений.
5. Ревью + мердж в `dev`.
6. Обновление документации при необходимости.
