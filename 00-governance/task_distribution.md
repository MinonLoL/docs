### 2.2 Backend разработка (Безручко А.В., Юркив А.А.)

#### Задача 2.1: Инициализация Flask-приложения
- **Статус:** ✅ Выполнено
- **Ответственный:** Безручко А.В. (Backend)
- **Описание:** Создание структуры проекта, конфигурация Flask
- **Артефакты:**
  - `app.py` — главное приложение Flask
  - `config.py` — конфигурация (DEBUG, DATABASE, SECRET_KEY)
  - `requirements.txt` — зависимости (Flask, SQLite3, pytest)
  - `.github/workflows/tests.yml` — CI/CD пайплайн

#### Задача 2.2: Модуль работы с БД и миграции
- **Статус:** ✅ Выполнено
- **Ответственный:** Безручко А.В. (Backend)
- **Описание:** Инициализация SQLite, создание таблиц, seed data
- **Артефакты:**
  - `db.py` — функции инициализации БД, создание таблиц
  - `pvz.db` — файл базы данных (SQLite)
  - `data/pvz.json` — справочник ПВЗ (50+ точек)
  - `data/schedule.json` — расписание работы
  - `data/operations.json` — примеры операций
  - `generate_realistic_data.py` — скрипт генерации тестовых данных

#### Задача 2.3: Система аутентификации и RBAC
- **Статус:** ✅ Выполнено
- **Ответственный:** Безручко А.В. (Backend)
- **Описание:** Реализация ролевой модели доступа (Оператор, Супервайзер, Аналитик)
- **Артефакты:**
  - `auth.py` — функции аутентификации, декораторы `@role_required`
  - `routes/auth.py` — endpoints для login/logout
  - Таблица `users` в БД с полями: login, password, role, pvz_id

#### Задача 2.4: CRUD операции для ПВЗ и расписания
- **Статус:** ✅ Выполнено
- **Ответственный:** Юркив А.А. (Backend)
- **Описание:** API endpoints для управления справочниками
- **Артефакты:**
  - `routes/pvz.py` — endpoints GET/POST/PUT/DELETE для ПВЗ
  - Endpoints:
    - `GET /api/pvz` — список всех ПВЗ
    - `GET /api/pvz/<id>` — детали ПВЗ
    - `GET /api/pvz/<id>/schedule` — расписание ПВЗ

#### Задача 2.5: API регистрации операций
- **Статус:** ✅ Выполнено
- **Ответственный:** Юркив А.А. (Backend)
- **Описание:** Endpoints для создания операций (прием, выдача, возврат)
- **Артефакты:**
  - `routes/operations.py` — endpoints для операций
  - Endpoints:
    - `POST /api/operations` — создание операции
    - `GET /api/operations` — список операций (с фильтрацией)
    - `GET /api/operations/<id>` — детали операции

#### Задача 2.6: Валидация операций
- **Статус:** ✅ Выполнено
- **Ответственный:** Безручко А.В. (Backend)
- **Описание:** Проверка 5 критериев валидации:
  1. Существование ПВЗ
  2. Корректный тип операции (in/out/return)
  3. Соответствие расписанию
  4. Время в рабочих часах
  5. Формат данных
- **Артефакты:**
  - Функции валидации в `routes/operations.py`
  - Логирование ошибок в таблицу `error_log`

#### Задача 2.7: Логирование ошибок валидации
- **Статус:** ✅ Выполнено
- **Ответственный:** Юркив А.А. (Backend)
- **Описание:** Ведение журнала ошибок с указанием причины
- **Артефакты:**
  - Таблица `error_log` в БД
  - Функции логирования в `db.py`
  - Endpoint `GET /api/error-log` для просмотра ошибок

### 2.3 Аналитика и отчетность (Безручко А.В., Углев М.А.)

#### Задача 3.1: API отчета по загрузке ПВЗ
- **Статус:** ✅ Выполнено
- **Ответственный:** Безручко А.В. (Backend)
- **Описание:** Расчет KPI: средняя загрузка, доля перегруженных интервалов
- **Артефакты:**
  - `routes/reports.py` — endpoints для отчетов
  - Endpoints:
    - `GET /api/report/load` — загрузка по часам
    - `GET /api/report/kpi` — ключевые показатели
  - Формула расчета: `load% = (операции_в_час / capacity_per_hour) * 100`

#### Задача 3.2: API генерации тепловой карты
- **Статус:** ✅ Выполнено
- **Ответственный:** Углев М.А. (Frontend) + Безручко А.В. (Backend)
- **Описание:** Визуализация загрузки по часам и дням недели
- **Артефакты:**
  - `routes/reports.py` — endpoint `GET /api/report/heatmap`
  - `docs/04-design/heatmap_dashboard.md` — описание дизайна
  - Возвращает матрицу: дни недели × часы с процентами загрузки

#### Задача 3.3: Экспорт данных в CSV
- **Статус:** ✅ Выполнено
- **Ответственный:** Углев М.А. (Frontend) + Безручко А.В. (Backend)
- **Описание:** Выгрузка данных с фильтрацией по периоду
- **Артефакты:**
  - `routes/reports.py` — endpoint `GET /api/report/export`
  - Параметры: date_from, date_to, pvz_id, operation_type
  - Возвращает CSV файл с полями: timestamp, pvz_id, operation_type, load%
