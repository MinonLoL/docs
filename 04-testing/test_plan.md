# План тестирования «ПВЗ Monitor»

## 1. Стратегия тестирования
Основной упор делается на автоматизированное функциональное тестирование API и проверку бизнес-правил валидации данных.

## 2. Виды тестирования
- **Unit-тесты:** Проверка отдельных функций (например, расчет загрузки в `reports.py`).
- **Интеграционные тесты:** Проверка взаимодействия API с базой данных SQLite.
- **Тестирование безопасности:** Проверка декораторов доступа (`@role_required`).

## 3. Ключевые тестовые сценарии

### 3.1. Валидация операций (Критический путь)
| ID | Сценарий | Ожидаемый результат |
| :--- | :--- | :--- |
| TC-VAL-01 | Создание операции с типом `in`/`out`/`return` | 201 Created |
| TC-VAL-02 | Создание операции с некорректным типом | 400 Bad Request |
| TC-VAL-03 | Операция в нерабочее время ПВЗ | 400 Bad Request |
| TC-VAL-04 | Операция для несуществующего ПВЗ | 400 Bad Request |

### 3.2. Авторизация и роли
| ID | Сценарий | Ожидаемый результат |
| :--- | :--- | :--- |
| TC-AUTH-01 | Доступ оператора к данным своего ПВЗ | Доступ разрешен |
| TC-AUTH-02 | Доступ оператора к чужому ПВЗ | 403 Forbidden |
| TC-AUTH-03 | Доступ аналитика к экспорту CSV | Доступ разрешен |

## 4. Инструментарий
- **Фреймворк:** `pytest`
- **Библиотеки:** `pytest-flask`, `pytest-mock`
- **Отчетность:** `pytest-html` (генерация HTML-отчетов)
